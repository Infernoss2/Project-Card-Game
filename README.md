# 🃏 Card Game System

## 📌 Overview
This project is a card game system implemented in Python using object-oriented programming principles.  
It simulates a full gameplay experience including players, card management, and dynamic game flow.

The system was designed with an emphasis on clean architecture, modularity, and scalability, allowing easy extension and modification of game logic.

---

## 🧠 Features
- Object-Oriented Design (Classes for Player, Card, Deck, Game)
- Dynamic game flow with multiple states
- Player interaction and decision-making
- Separation of concerns between game logic and player logic
- Scalable and maintainable code structure

---

## 🎮 Game Rules (Simplified)

- Each player receives a set of cards at the beginning of the game.
- Some cards are placed face-down, and others are visible to the player.
- Players take turns playing cards according to the game rules.
- A player must play a valid card based on the current state of the game.
- If a player cannot play a valid card, they may need to draw or take cards.
- The objective is to get rid of all your cards before the other players.

> Note: The game includes strategic decision-making regarding which cards to play and when.
